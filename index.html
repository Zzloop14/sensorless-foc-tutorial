按这个顺序读图

不要从左到右硬读，FOC 是闭环系统。先分清功率链、采样链、角度链和控制链。

1. 功率链 SVPWM -> driver -> 三相逆变 -> PMSM，这是最终把控制量变成三相电压的路径。
2. 电流反馈链 Ia/Ib/Ic -> Clarke -> Iα/Iβ -> Park -> Id/Iq，这是电流环能闭上的基础。
3. 无感角度链 Iα/Iβ + Uα/Uβ -> 滑模观测器 -> θr/ω，估出的角度决定 d/q 轴摆在哪里。
4. 控制输出链 速度 PI -> Iq_ref，Id_ref=0，两个电流 PI 输出 Ud/Uq，再反 Park 或矢量合成送入 SVPWM。

Profiling

怎么判断每段代码真实耗时

快环总周期由 PWM_FREQUENCY = 20000 决定，所以一拍只有 50 us。网页里的分段时间只是预算示意，真正能用于判断风险的是实测值：测 ADC 中断总耗时，再逐段测 m_phase_current_calculate()、m_observer_execute()、m_foc_algorithm_execute() 和 m_svpwm_generate()。

GPIO 示波器法 最直观。函数前拉高 GPIO，函数后拉低 GPIO，示波器看到的高电平宽度就是耗时。
GPT 计数法 适合不用示波器时做数值记录。若计数时钟 120 MHz，则 time_us = delta_count / 120。
DWT 周期计数法 适合 Cortex-M 内核精细测量。若 CPU 120 MHz，则 time_us = cycles / 120。

GPIO 实测波形示意

一个测量引脚只包住一个函数；多个函数建议换不同 GPIO，或者分多次烧录测量，避免测量代码本身干扰过大。

坐标变换

从三相电流到 d/q 电流

这个实验区把 abc -> alpha/beta -> d/q 过程做成了可视化。你只需要调整电流矢量角度和转子角度，就能看到为什么 Id 表示磁链分量、Iq 表示转矩分量。

第一层：ia / ib / ic 是真实绕组电流 它们来自 ADC 采样和零偏修正，是 A/B/C 三相线圈里的物理电流。三相无中线电机满足 ia + ib + ic = 0，所以三个量里实际只有两个自由度。
第二层：iα / iβ 是静止平面的合成矢量 Clarke 不是产生新电流，而是把三相电流“合成为一个箭头”。这个箭头固定在定子坐标系里，α 轴通常和 A 相轴对齐。
第三层：Id / Iq 是转子视角下的投影 Park 把观察角度换到转子上。Id 是沿转子磁链方向的电流，Iq 是垂直磁链方向的电流，主要决定转矩。

Clarke：三相合成一个静止矢量

A/B/C 三相轴互差 120°。在平衡三相里，第三相可以由另外两相推出，因此可以等价成 αβ 平面上的一个电流箭头。

Park：把静止矢量投影到转子坐标

同一个 iα/iβ，转子角度不同，投到 d/q 轴上的结果就不同。FOC 控制的是这个投影结果，而不是直接控制三相电流波形。

各个量之间的关系

三相物理量 ia, ib, ic: 三相绕组电流 约束: ia + ib + ic = 0
Clarke 输出 iα = ia iβ = (ia + 2 * ib) / sqrt(3)
Park 输出 Id = iα * cosθ + iβ * sinθ Iq = -iα * sinθ + iβ * cosθ

为什么 FOC 一定要这样变换

三相电流是交流量，直接用 PID 控制会遇到一个问题：目标值和实际值都在随转子位置变化。Park 变换后，如果角度正确，稳定转矩对应的 Id/Iq 会接近直流量，这样电流环就能像控制普通直流量一样工作。

目标： Id_target = 0 Iq_target = 转矩电流 含义： Id 越接近 0，电流越少浪费在励磁方向 Iq 越稳定，输出转矩越稳定

所以无感 FOC 的角度估算很关键：角度错了，Park 投影就错，Id 和 Iq 会互相串扰，表现为电流大、转矩小、噪声大或启动失败。

SMO

滑模观测器与参数交互

当前工程实现的是带边界层的离散滑模观测器。左侧参数会驱动右侧的饱和特性曲线，并给出对抖振、收敛和角度噪声的定性判断。判断逻辑基于这份工程的参数范围，不是抽象教材默认值。

观测器要解决什么 无感 FOC 没有编码器，所以程序不知道真实转子角度。观测器用已知的 Uα/Uβ、实测的 Iα/Iβ 和电机 R/L 模型，反推出反电势 Eα/Eβ，再由反电势得到 θe。
滑模观测器的核心动作 先用模型预测电流 i_hat，再看预测电流和真实电流的误差 s=i_hat-i。误差大就强力修正，误差小就线性修正，修正量 z 经过低通后近似反电势。
为什么适合这个工程 SMO 计算量小、适合 Q15 定点、对参数误差比纯模型积分更抗干扰。代价是低速反电势弱、K/Smax 不合适会抖振，所以工程里用了边界层和两级低通。

从电机模型到离散观测器

连续模型中的未知量是反电势 E。SMO 通过让估计电流贴近真实电流，把未知反电势转换成可观测的滑模修正量 z。

常见无感观测器对比

反电势过零 / 反电势直接法 优点: 简单，适合方波/BLDC 缺点: 低速弱，FOC 正弦控制下角度连续性不够
Luenberger / 全阶或降阶观测器 优点: 模型清晰，输出平滑 缺点: 更依赖 R/L/Ke 参数，增益设计复杂
MRAS 模型参考自适应 优点: 可自适应估计速度/参数 缺点: 推导和稳定性设计更重，调参链路长
EKF 扩展卡尔曼滤波 优点: 抗噪声强，可融合多状态 缺点: 计算量大，矩阵和噪声协方差调试成本高
高频注入 HFI 优点: 零速/低速也能估角 缺点: 需要电机凸极性，噪声和实现复杂度更高

本工程为什么选 SMO

这份工程的目标是 20 kHz 快环内完成无感 FOC，平台适合定点运算。SMO 不需要矩阵运算，只依赖加减乘、饱和函数和低通滤波，计算量可控；同时它直接利用电流误差闭环修正，比单纯开环积分模型更能抗参数误差和采样噪声。

注意：低速时反电势本身很小，任何反电势类观测器都会困难。工程通过预定位、开环强拖、切闭环判断解决零速启动问题，而不是一上电就靠 SMO 闭环。

电机模型: dI/dt = -R/L * I + 1/L * (U - E) 离散形式: i_hat[n] = F*i_hat[n-1] + G*u[n-1] - G*z[n-1] - G*e_hat[n-1]

变量逐项说明

Iα/Iβ 来源: Clarke 变换后的真实电流，来自 m_foc_unit.coordinate.q15_i_alpha/beta 用途: 作为观测器校正参考，和 i_hat 比较得到误差 s
Uα/Uβ 来源: 上一拍 SVPWM/电压矢量分解得到的 q15_u_last_alpha/beta 用途: 作为电机模型输入，参与下一拍估计电流计算
F / G 来源: F=1-Ts*R/L，G=Ts/L，在 m_parameter.h 转成 Q15 用途: 把连续电机方程离散化，决定估计电流的模型动态
i_hat 来源: m_observer_estimate_current() 用 F/G/U/Z/E 递推得到 用途: 和真实电流做差，产生滑模面 s=i_hat-i
Smax 来源: OBSERVER_S_MAX_VALUE=170，边界层宽度 用途: 决定误差多大进入饱和区；越小越硬，越大越平滑
K 来源: OBSERVER_K_SLIDE=30000，滑模增益 用途: 决定 z 最大修正能力；过小跟不上，过大易抖振
zα/zβ 来源: z=K*sat(s/Smax)，由 m_observer_z_calculate() 计算 用途: 一边反馈修正估计电流，一边经过低通提取反电势
Kafc 来源: 根据估计速度/角度增量自适应计算，写入 m_obs_unit.q15_kafc 用途: 作为两级低通系数，速度越高通常允许更高带宽
Eα/Eβ 来源: zα/zβ 经过两级 math_q15_digital_lpf() 用途: 作为反电势矢量，用于 atan2(Eβ,Eα) 求角度
θe 来源: m_sensorless_theta_e_calculate() 中 atan2(Eβ,Eα) 后再做 90°补偿 用途: 确定 d/q 坐标轴位置，送给 Park/FOC/SVPWM 相关角度链路
z = K * sat(s / Smax) if s > Smax: z = +K if s < -Smax: z = -K else: z = K * s / Smax
离散电流估算 i_hat[n] = F*i_hat[n-1] + G*u[n-1] - G*z[n-1] - G*e_hat[n-1]
代码位置 src/motor/m_observer.c:80, 102, 144
角度出口 theta_e = atan2(e_beta_final, e_alpha_final)，位于 src/motor/m_rotor_angle.c:127
